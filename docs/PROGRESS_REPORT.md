# Progress Report

## Current Status

The project is now a working Python package for generating a knowledge-guided synthetic
benchmark dataset.

Core pipeline status:

- FNDDS survey JSON can be loaded into canonical food and ingredient objects.
- Synthetic label transformations run through the operator engine.
- Ingredient declarations, allergen declarations, claims, and Nutrition Facts are rendered.
- Final rendered samples are validated after rendering.
- Dataset directories can be written with canonical data, ground truth, structured labels,
  rendered text, operators, validation reports, and metadata.
- Train/validation/test split files can be generated by FDC ID.
- Reference-only regex parsing and dictionary mapping baselines exist for development smoke checks.
- GitHub Actions CI is configured for linting, type checking, and tests.

Validation and semantic fixes:

- Missing nutrient values are no longer treated as measured zero when generating claims.
- Allergen detection now includes sesame and uses boundary-aware matching.
- Outdated or over-broad allergen assumptions were corrected, including coconut/tree-nut and
  cream-of-tartar false positives.
- LLM fallback now returns the original rendered label text instead of the full prompt.
- Validation now runs after template rendering and optional LLM refinement.
- Ground truth is populated with canonical mappings, target mappings, ingredient amounts,
  ingredient fractions, transformation history, and validation reports.
- Mapping metrics now use the same target namespace as the built-in dictionary baseline.
- Parsing metrics normalize case and handle empty inputs safely.
- Dataset CSV writing uses the standard CSV writer.
- Dataset splitting handles small datasets and zero ratios more predictably.
- Difficulty is based on operators that actually affect output.

Real-data calibration:

- Ingredient labels are initialized in descending ingredient-fraction order.
- Ingredient-order validation checks label weight order and two-percent grouping rather than raw
  FNDDS sequence order.
- Nutrition Facts validation uses the same serving-size and FDA-rounding path as generation.
- Claim validation distinguishes `SATURATED FAT FREE` from total-fat `FAT FREE`.
- Duplicate claims are removed.
- Allergen validation uses structured allergen declarations before parsing text.
- Over-broad prohibited-term checks for ingredient descriptors such as `fresh` and `imitation`
  were removed.

## Problems Encountered

Repository setup:

- The project directory originally did not have its own `.git` directory. Git resolved upward to
  the user home directory, which made normal `git status` noisy and unsafe.
- A separate Git repository was initialized inside `Synthetic_benchmark_db` before pushing to
  GitHub.

CI dependency layout:

- The package depends on `nusol` through a local sibling path, `../NuSol-T`.
- CI therefore checks out `CedarJin/NuSol-T` beside this repository to match the expected local
  path layout.

Validation ordering:

- Text validation originally ran before rendering, so rules inspecting rendered text could miss
  final-output issues.
- The generation flow was changed to render first, then validate the final sample.

Ground truth completeness:

- `ground_truth.json` originally contained mostly the canonical food and did not populate the fields
  needed for benchmark evaluation.
- Ground truth generation now fills mappings, ingredient amounts/fractions, transformation history,
  and validation results.

Claims:

- Missing nutrients were previously treated as zero, causing invalid zero-based claims.
- Some validation failures came from comparing claims with a different serving-size/rounding path
  than generation.
- `SATURATED FAT FREE` was being interpreted as total-fat `FAT FREE`.

Allergens:

- Sesame was missing.
- Some substring matches caused false positives.
- `CRUSTACEAN SHELLFISH` text could be parsed as `fish` during validation.
- Structured allergen data now takes priority over text fallback.

Ingredient order:

- Real FNDDS ingredient sequence does not always correspond to descending ingredient weight.
- The benchmark label should follow declared fraction order, so both generation and validation were
  recalibrated around fractions rather than raw FNDDS sequence.

Nutrition Facts:

- Validation initially compared against unrounded values while generation emitted rounded values.
- Validation now uses the same rounding path as generation.

Prohibited terminology:

- Broad checks for `fresh` and `imitation` flagged legitimate ingredient descriptors, such as
  fresh pork or imitation vanilla extract.
- Those broad terms were removed from the global prohibited-term list.

Pilot dataset inspection:

- The 200-sample `benchmark_v0.1` pilot exposed an invalid generic-name substitution where
  `Butter oil, anhydrous` could become `LETTUCE`.
- Ambiguous single-word generic mappings were removed or made more specific.
- `GenericNameOperator` now only accepts single-word substitutions when the declared name also
  contains the target ingredient category.
- Source claims are now limited to suitable positive nutrients instead of generating claims such as
  `EXCELLENT SOURCE OF SODIUM`, `EXCELLENT SOURCE OF TOTAL LIPID (FAT)`, or
  `EXCELLENT SOURCE OF FATTY ACIDS`.

## Remaining Issues

Under the current project boundary, parser and mapper quality are not blocking issues for this
repository. Those are downstream NIP-M responsibilities.

The main remaining work is dataset release readiness:

- Add a dataset card covering source data, generation settings, validation coverage, and known
  limitations.
- Add regression fixtures from representative generated samples.
- Decide whether `../NuSol-T` should remain a local path dependency or become a package/Git
  dependency.
- Re-review FDA rule coverage before treating the benchmark as publication-ready.
- Stabilize the `ground_truth.json` contract for NIP-M handoff.
- Package a release dataset artifact with manifest, schema version, and checksums.
